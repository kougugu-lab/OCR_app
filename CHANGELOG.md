# CHANGELOG

本アプリケーションの変更履歴を記録します。

## [2026-04-05] v1.2.0 - 共通UI部品の集約とロギング機能の統合

### 追加 (Added)
- **ロギング機能の統合**: デイリーログ出力 (`results/logs/app_YYYYMMDD.log`) を実装し、初期化・カメラ接続・検査結果の追跡（履歴および詳細）を強化。
- **UI部品専用モジュールの新設**: `modules/widgets.py` を作成し、共通UIコンポーネント（ToolTip, LEDIndicator, ToggleSwitch, HybridSlider, ScrollableFrame, HelpWindow）を整理・集約。
- **カメラ接続確認**: アプリ起動時にカメラの接続状態をチェックし、失敗した場合はユーザーへエラー内容をダイアログ通知する機能を追加。

### 変更 (Changed)
- **OCRテストの最適化**: 設定画面での「OCRテスト」実行時に、メインアプリのOCRエンジンインスタンスを再利用するように改善。テストの起動速度向上とメモリリソースの節約を実現。
- **ディスククリーンアップの拡張**: 保存フォルダの容量上限チェック時に、画像だけでなくログファイル（.log, .csv）も削除対象に含めることができるロジックへ更新。
- **コードのリファクタリング**: `settings_dialog.py` からUI部品定義を分離し、モジュール構成を最適化。

### 修正 (Fixed)
- **検査ループの制御修正**: トリガー有効/無効モードの切り替えおよび、それぞれのモードにおける判定ロジックがログ出力を伴い、正しく動作するように復元・修正。
- **例外処理の強化**: アプリケーション全体の初期化時およびメインループ内でのエラー捕捉とログ出力を追加。

## [2026-03-29] v1.1.0 - UI標準化およびファイル命名規則の刷新

### 追加 (Added)
- **GPIOバリデーション**: 設定画面にて GPIO ピンの重複割り当てを禁止するチェック機能を追加 (§137)。
- **技術詳細ツールチップ**: 設定画面の各項目に規約に基づいた詳細な技術解説（ツールチップ）を追加 (§389, §312)。
- **信頼度記録**: OCR 判定結果の確信度（Confidence）をイメージファイル名とCSVログに記録する機能を追加。

### 変更 (Changed)
- **ディレクトリ構成の標準化**: 保存先をアプリルートの `results/` に統一し、サブフォルダ名を大文字 (`OK`, `NG`, `SKIP`, `REC`) へ変更。`BASE_DIR` の計算を修正 (§8-3)。
- **ファイル命名規則の刷新**: 画像保存名を `{判定}_{正解文字}_{カメラ}_{日付時刻14桁}_{信頼度}.jpg` に統一。
- **CSVログ形式の統一**: ヘッダーを日本語表記 (`日時, 正解文字, トリガー, カメラ名, 判定結果, 信頼度, 詳細内容`) に変更。
- **ステータスバー配色**: メイン画面ヘッダーの背景色を、待機・検査中・OK・NG・SKIP の各状態に応じて規約通りに変化するよう修正 (§4-2-1, §6-3)。
- **フォルダ構成の刷新**: コアモジュールを `modules/` フォルダへ、検証スクリプトを `tests/` フォルダへ移動し、root をクリーンアップ。
- **インポートの統一**: 全モジュール内のインポートをパッケージ参照および相対インポート (`from .xxx import ...`) へ更新。
- **軽量化・制御**: 詳細設定表示中のメイン画面更新停止 (§5-143) および、「ブザー停止」ボタンによる信号リセット機能を実装。

### 修正 (Fixed)
- **バグ修正 (SyntaxError)**: `modules/settings_dialog.py` 内の Python 構文エラー（セミコロン後の複合文）を修正。
- **バグ修正 (ModuleNotFoundError)**: パッケージ構成変更に伴う相対インポート指定ミスを修正。
- **バグ修正 (ValueError)**: 設定画面の数値項目（フォーカス値等）が空の場合に発生するエラーを修正。

## [2026-03-28] v1.0.1 - 照合ロジックの拡充とヘルプ機能の追加
### 追加 (Added)
- 正解カメラで特定の文字列（治具名や空パレット用文字列など）を読み取った際に、検査をSKIPして次のトリガーを待機する機能を追加（判定スキップ）。
- 「部分一致（検出文字に含まれる）」か「完全一致」かを設定画面で切り替え可能にする機能を追加（照合ロジック選択）。
- プレビュー映像のFPS設定機能を追加し、描画負荷を低減（リフレッシュレート調整）。
- メイン画面の「？」ボタンから詳細な操作ガイドを表示する機能を追加（ヘルプ拡充）。
- アプリの概要、設定、操作手順をまとめた `README.md` を新規作成。

### 変更 (Changed)
- **UI改善**: ヘッダーのステータス配色を共通要件 (§4-2-1) に準拠（待機時：パネル色、動作時：アクセント色）。
- **UI改善**: GPIOピンの指定範囲を BCM 有効範囲（2-27）に制限 (§181)。
- **UI改善**: 判定結果の画像ファイル命名規則を `{判定}_{モード}_{カメラ}_{日付時刻14桁}.jpg` に変更。
- **フォルダ構成の標準化**: `common_ui_requirements.md` に準拠し、保存先を `results/images/ok|ng|skip`, `results/logs/`, `results/csv/` に再編。
- **保存処理**: 検査結果（画像・CSVログ）の保存パスを新しい構成に合わせて修正。
- **設定初期値**: `constants.py` に新機能用のデフォルト設定値を追加。
