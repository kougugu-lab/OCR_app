# OCR 照合システム (OCR_app)

**バージョン**: 1.2.0  
**最終更新**: 2026-04-05

AIカメラを使用した、文字(OCR)照合検査アプリケーションです。
外部トリガーを契機として正解カメラと検査カメラの映像を比較し、文字の一致・不一致を判定します。

## 機能概要

* **2カメラインスペクション:** 正解カメラ（基準）と検査カメラ（対象）の映像を同時に処理し、文字を照合します。
* **トリガー制御:** GPIO経由での外部トリガー入力に対応しており、PLC等と連携した自動検査が可能です。連続検査モードにも対応しています。
* **柔軟な判定機能:** 判定ロジックとして「完全一致」および「部分一致（検査文字が正解文字を含む）」の選択が可能です。
* **スキップパターン:** 特定の文字列を読み取った際に、判定をスキップして次のトリガーを待機する設定が可能です。
* **リフレッシュレート制御:** パフォーマンス最適化のため、プレビュー映像のFPSを調整可能です。
* **NG履歴表示とエビデンス保存:** 読み取り結果やNGの履歴を一覧で確認でき、結果は自動的に画像およびCSVとして保存されます。

## 出力フォルダ構成

結果データは以下のディレクトリ構成で保存されます。

```
results/
  images/
    ok/      # OK判定時の画像
    ng/      # NG判定時の画像
    skip/    # スキップ判定時の画像
    rec/     # 写真撮影モード用の画像 (拡張時用)
  logs/      # アプリケーションの動作ログ
  csv/       # 検査履歴 (日別のCSVファイル)
```

## 設定方法

`config.json` に設定が保存されます。アプリケーション内の「詳細設定」ボタンから変更が可能です。
主要な項目は以下の通りです。

* **camera_ref_id**, **camera_insp_id**: 正解カメラ、検査カメラのUSBインデックス
* **match_mode**: `partial` (部分一致) または `exact` (完全一致)
* **skip_patterns**: 検査をSKIPするパターンのリスト (例: `["EMPTY", "JIG"]`)
* **preview_fps**: メイン画面プレビューの更新頻度 (1〜30fps)
* **mode_trigger**: `true` の場合、GPIOトリガー待機モード。`false` の場合、自動連続モード。

## 操作手順

1. カメラとGPIOインターフェース(Raspberry Pi等)を接続してアプリを起動します。
2. 画面右下の「詳細設定」を開き、カメラのインデックスを選択します。
3. （必要に応じて）「正解カメラ設定」および「検査カメラ設定」の「検出エリア」をドラッグしてOCR対象範囲を指定します。
4. 設定を保存するとメイン画面に戻ります。
5. 「トリガーモード」が有効な場合、GPIOからのトリガー入力時に正解・検査カメラの文字を照合し、結果を表示（および保存）します。
6. 不明点があれば、画面右上の「？」ボタンでヘルプダイアログを開き確認してください。

## 開発環境と留意点

* WindowsおよびLinux（Raspberry Pi OS等）での実行をサポートしています。
* OCR推論エンジンには `rapidocr_onnxruntime` を使用します。
* 外部ライブラリ (`cv2`, `PIL`, `tkinter` など) の環境構築が必要です。
